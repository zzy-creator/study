### <font style="color:rgb(38, 38, 38);">js异步</font>
将需要网络请求、文件读写等耗时操作的函数设置为异步

#### 基本
<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">async</font>表示异步函数，会异步执行它

+ 异步函数返回<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">promise</font>，promise.then/catch/finally是异步函数的回调函数，可以在回调函数中使用异步函数的返回值；使用await，可以直接获得异步函数的返回值（若异步函数执行出错，则会抛出error，需要trycatch）
+ 同步函数中调用async函数，会异步执行他，返回一个promise，若需要在async函数执行完成后进行操作，则在promise.then/catch中进行
+ async函数内部同步顺序执行，若遇到异步函数，可用<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">await</font>等他。**await可能抛error的话，要用try catch**

#### 原理
详见：[https://juejin.cn/post/6844903692752142343](https://juejin.cn/post/6844903692752142343)



1. <font style="color:rgb(102, 102, 102);">JS 是单线程的，只有一个主线程</font>
2. <font style="color:rgb(102, 102, 102);">函数内的代码从上到下顺序执行，遇到被调用的函数先进入被调用函数执行，待完成后继续执行</font>
3. <font style="color:rgb(102, 102, 102);">遇到异步事件，借助浏览器、nodejs等宿主环境另开一个线程，主线程继续执行，待结果返回后，执行回调函数</font>

<img src="https://intranetproxy.alipay.com/skylark/lark/0/2023/png/88656377/1696748227521-f4306b0b-2ccb-44fe-944e-c04309047f5c.png" width="403" title="" crop="0,0,1,1" id="u5c92394e" class="ne-image">

+ js<font style="color:rgb(37, 41, 51);">只有一个主线程和一个调用栈（</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">call stack</font><font style="color:rgb(37, 41, 51);">）</font>
+ <font style="color:rgb(37, 41, 51);">同步任务：在主线程的调用栈上排队执行，只有前一个任务执行完毕，才能执行后一个任务。</font>
+ <font style="color:rgb(37, 41, 51);">异步任务：首先交给浏览器的其他模块进行处理，处理完成后将回调函数放入</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">"任务队列"（task queue）</font><font style="color:rgb(37, 41, 51);">， </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">"任务队列"</font><font style="color:rgb(37, 41, 51);">通知主线程后，主线程执行。</font>
    - <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">macro-task（宏任务）</font>大概包括：script(整体代码), setTimeout, setInterval, setImmediate（NodeJs）, I/O, UI rendering。
    - <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">micro-task（微任务）</font><font style="color:rgb(37, 41, 51);">大概包括: process.nextTick（NodeJs）, </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">Promise</font><font style="color:rgb(37, 41, 51);">, Object.observe(已废弃), MutationObserver(html5新特性)</font>
    - <font style="color:rgb(37, 41, 51);">不同任务会进入到不同的任务队列</font>
+ <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">事件循环（总的执行流程）：</font>
    1. <font style="color:rgb(37, 41, 51);">调用栈顺序调用任务</font>
    2. <font style="color:rgb(37, 41, 51);">当调用栈发现异步任务时，将异步任务交给其他模块处理，自己继续进行下面的调用</font>
    3. <font style="color:rgb(37, 41, 51);">异步执行完毕，异步模块将任务推入任务队列，并通知调用栈</font>
    4. <font style="color:rgb(37, 41, 51);">调用栈清空后，将执行任务队列里的微任务队列</font>
    5. <font style="color:rgb(37, 41, 51);">微任务队列执行完成后，执行一个宏任务队列</font>
    6. <font style="color:rgb(37, 41, 51);">执行结束后，检查有无微任务，有则执行；无则执行一个宏任务队列</font>
    7. <font style="color:rgb(37, 41, 51);">循环至没东西可执行</font>

### <font style="color:rgb(73, 73, 73);">浏览器模型</font>
#### <font style="color:rgb(73, 73, 73);">浏览器环境</font>
<font style="color:rgb(73, 73, 73);">浏览器的核心是两部分：渲染引擎和 JavaScript 解释器（又称 JavaScript 引擎）。</font>

<font style="color:rgb(73, 73, 73);"></font>

<font style="color:rgb(73, 73, 73);">渲染引擎的主要作用是，将网页代码渲染为用户视觉可以感知的平面文档。</font>

<font style="color:rgb(73, 73, 73);">不同的浏览器有不同的渲染引擎。</font>

**<font style="color:rgb(73, 73, 73);">渲染引擎处理网页</font>**<font style="color:rgb(73, 73, 73);">，通常分成四个阶段。</font>

1. <font style="color:rgb(73, 73, 73);">解析代码：HTML 代码解析为 DOM，CSS 代码解析为 CSSOM（CSS Object Model）。</font>
2. <font style="color:rgb(73, 73, 73);">对象合成：将 DOM 和 CSSOM 合成一棵渲染树（render tree）。</font>
3. <font style="color:rgb(73, 73, 73);">布局：计算出渲染树的布局（layout）。</font>
4. <font style="color:rgb(73, 73, 73);">绘制：将渲染树绘制到屏幕。</font>

<font style="color:rgb(73, 73, 73);">以上四步并非严格按顺序执行，往往第一步还没完成，第二步和第三步就已经开始了。所以，会看到这种情况：网页的 HTML 代码还没下载完，但浏览器已经显示出内容了。</font>

<font style="color:rgb(73, 73, 73);">渲染树转换为网页布局，称为“布局流”（flow）；布局显示到页面的这个过程，称为“绘制”（paint）。它们都具有阻塞效应，并且会耗费很多时间和计算资源。</font>

**<font style="color:rgb(73, 73, 73);">重流和重绘</font>**<font style="color:rgb(73, 73, 73);">并不一定一起发生，重流必然导致重绘，重绘不一定需要重流。比如改变元素颜色，只会导致重绘，而不会导致重流；改变元素的布局，则会导致重绘和重流。</font>

<font style="color:rgb(73, 73, 73);">作为开发者，应该尽量</font>**<font style="color:rgb(73, 73, 73);">设法降低重绘的次数和成本</font>**<font style="color:rgb(73, 73, 73);">。比如，尽量不要变动高层的 DOM 元素，而以底层 DOM 元素的变动代替</font>

<font style="color:rgb(73, 73, 73);"></font>

<font style="color:rgb(73, 73, 73);">JavaScript 引擎的主要作用是，读取网页中的 JavaScript 代码，对其处理后运行。</font>

<font style="color:rgb(73, 73, 73);"></font>

#### <font style="color:rgb(73, 73, 73);">window对象</font>
<font style="color:rgb(73, 73, 73);">浏览器里面，</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">window</font><font style="color:rgb(73, 73, 73);">对象（注意，</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">w</font><font style="color:rgb(73, 73, 73);">为小写）指当前的浏览器窗口。</font>

<font style="color:rgb(73, 73, 73);">它也是当前页面的顶层对象，所有其他对象都是它的下属。</font>

<font style="color:rgb(73, 73, 73);">一个变量如果未声明，那么默认就是顶层对象的属性。</font>

<font style="color:rgb(73, 73, 73);"></font>

#### <font style="color:rgb(73, 73, 73);">Cookie</font>
<font style="color:rgb(73, 73, 73);">用户首次向服务器请求时，服务器生成一个Cookie来区分用户。Cookie是服务器保存在浏览器的一小段文本信息（小于4KB）。浏览器每次向服务器发出请求，就会自动附上这段信息。</font>

<font style="color:rgb(73, 73, 73);">Cookie 由 HTTP 协议生成，也主要是供 HTTP 协议使用。</font>

<font style="color:rgb(73, 73, 73);">Cookie 不是一种理想的客户端存储机制。</font>

+ <font style="color:rgb(73, 73, 73);">它的容量很小（4KB），缺乏数据操作接口，而且会影响性能</font>
+ <font style="color:rgb(73, 73, 73);">客户端存储建议使用 Web storage API 和 IndexedDB</font>
+ <font style="color:rgb(73, 73, 73);">只有那些每次请求都需要让服务器知道的信息，才应该放在 Cookie 里面。</font>

<font style="color:rgb(73, 73, 73);">Cookie的使用场景主要如下：</font>

+ <font style="color:rgb(73, 73, 73);">对话（session）管理：保存登录状态、购物车等需要记录的信息。</font>
+ <font style="color:rgb(73, 73, 73);">个性化信息：保存用户的偏好，比如网页的字体大小、背景色等等。</font>
+ <font style="color:rgb(73, 73, 73);">追踪用户：记录和分析用户行为。</font>

<font style="color:rgb(73, 73, 73);"></font>

#### <font style="color:rgb(73, 73, 73);">AJAX</font>
<font style="color:rgb(73, 73, 73);">AJAX指通过 JavaScript 的异步HTTP通信，js脚本发出http请求，接受服务器返回的数据来更新网页</font>

+ <font style="color:rgb(51, 51, 51);">XML 被设计用来传输和存储数据，其焦点是数据的内容。</font>
+ <font style="color:rgb(51, 51, 51);">HTML 被设计用来显示数据，其焦点是数据的外观。</font>

<font style="color:rgb(73, 73, 73);">现在，服务器返回的都是 JSON 格式的数据，XML 格式已经过时了</font>

<font style="color:rgb(73, 73, 73);"></font>

#### <font style="color:rgb(73, 73, 73);">同源政策</font>
<font style="color:rgb(73, 73, 73);">浏览器安全的基石是“同源政策”</font>

<font style="color:rgb(73, 73, 73);">  
</font><font style="color:rgb(73, 73, 73);">所谓“同源”指的是“三个相同”：</font>

+ 协议相同
+ 域名相同（www.example.com/dir 和 www.example.com/dir1域名相同）
+ 端口相同（虽然但是，浏览器并<font style="color:rgb(73, 73, 73);">没有遵守这条规定</font>）



目前，如果非同源，共有三种行为受到限制：

+ <font style="color:rgb(73, 73, 73);">无法读取非同源网页的 Cookie、LocalStora和 IndexedDB。</font>
+ <font style="color:rgb(73, 73, 73);">无法接触非同源网页的 DOM。</font>
+ <font style="color:rgb(73, 73, 73);">无法向非同源地址发送 AJAX 请求（可以发送，但浏览器会拒绝接受响应）（现在大部分服务器都支持CORS跨源资源共享通信，所以是可以向非同源的发HTTP请求的）。</font>

<font style="color:rgb(73, 73, 73);"></font>

#### <font style="color:rgb(73, 73, 73);">Storage接口</font>
<font style="color:rgb(73, 73, 73);">Storage 接口用于脚本在浏览器保存数据。两个接口：</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">window.sessionStorage</font><font style="color:rgb(73, 73, 73);">和</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">window.localStorage</font><font style="color:rgb(73, 73, 73);">。</font>

+ <font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">sessionStorage</font><font style="color:rgb(73, 73, 73);">保存的数据用于浏览器的一次会话（session），当会话结束（通常是窗口关闭），数据被清空</font>
+ <font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">localStorage</font><font style="color:rgb(73, 73, 73);">保存的数据长期存在，下一次访问该网站时也仍存在</font>

<font style="color:rgb(73, 73, 73);">保存的数据都以“键值对”的形式存在。</font>

<font style="color:rgb(73, 73, 73);">所有的数据都以文本格式（字符串）保存。</font>

<font style="color:rgb(73, 73, 73);">每个域名的存储上限视浏览器而定，大部分是几MB</font>

<font style="color:rgb(73, 73, 73);">某个网页存入的数据，只有同域下的网页才能读取，如果跨域操作会报错。</font>

<font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">Location</font><font style="color:rgb(73, 73, 73);">对象是浏览器提供的原生对象，提供 URL 相关的信息和操作方法。通过</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">window.location</font><font style="color:rgb(73, 73, 73);">和</font><font style="color:rgb(218, 16, 57);background-color:rgb(244, 244, 244);">document.location</font><font style="color:rgb(73, 73, 73);">属性，可以拿到这个对象。</font>

## 响应式依赖
<font style="color:rgb(0, 0, 0);">"响应式依赖"是指：</font>**<font style="color:rgb(0, 0, 0);">响应式数据变化时，依赖于该数据的相关代码会自动更新</font>**



<font style="color:rgb(0, 0, 0);">常见的实现响应式依赖的方式包括：</font>

+ <font style="color:rgb(0, 0, 0);">观察者模式：通过定义观察者对象，当数据发生变化时，通知所有观察者进行更新。</font>
+ <font style="color:rgb(0, 0, 0);">发布-订阅模式：通过发布者发布事件或消息，订阅者注册对应的事件处理函数，在数据变化时接收到对应的事件通知。</font>
+ <font style="color:rgb(0, 0, 0);">响应式框架：如Vue.js、React等，提供了内置的响应式系统，通过使用特定的语法或API来声明数据和依赖的关系，框架会自动追踪依赖和处理更新。</font>

### 内存泄漏
**<font style="color:rgb(139, 139, 139);">内存泄漏（Memory Leak）</font>**<font style="color:rgb(139, 139, 139);">：</font><font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">  
</font><font style="color:rgb(139, 139, 139);">当程序不再需要某个对象时，仍然由于某些引用存在，导致 </font>**<font style="color:rgb(139, 139, 139);">垃圾回收器（GC）无法释放该对象占用的内存</font>**<font style="color:rgb(139, 139, 139);">，从而造成内存浪费。</font>

<font style="color:rgb(139, 139, 139);"></font>

可能导致内存泄漏的情况：

+ console.log
    - 使用devtools排查内存泄漏:[https://juejin.cn/post/7185128318235541563](https://juejin.cn/post/7185128318235541563)
    - [https://blog.rexskz.info/getting-to-bottom-will-console-log-cause-memory-leak-when-devtools-is-off.html?cc_lang=zh](https://blog.rexskz.info/getting-to-bottom-will-console-log-cause-memory-leak-when-devtools-is-off.html?cc_lang=zh)
    - 结论：不管开不开devtools，console.log都会导致内存泄漏。所以生产环境应该去掉console.log
+ 定时器用完了没有清除，那每次执行都会多一个定时器的内存占用，这就是内存泄漏
+ 元素从 dom 移除了，但是还有一个变量引用着他，这样的游离的 dom 元素也不会被回收。每执行一次代码，就会多出游离的 dom 元素的内存，这也是内存泄漏
+ 闭包引用了某个变量，这个变量不会被回收，如果这样的闭包比较多，那每次执行都会多出这些被引用变量的内存占用。这样引用大对象的闭包多了之后，也会导致内存问题
+ 全局变量，这个本来就不会被 GC，要注意全局变量的使用



事件和监听一定要清理

`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">addEventListener</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">、</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">on</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">、</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">subscribe</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">、</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(239, 246, 255);">setInterval</font>`<font style="color:rgb(0, 0, 0);">一定要清理，不然会有内存泄露的问题。尤其是绑定在window或者document对象上时。</font>

<font style="color:rgb(0, 0, 0);">例子：</font>

```javascript

  const handler = () => {
    setCount(prev => prev + 1); // 修改 state
  };
```

+ <font style="color:rgb(13, 18, 57);">组件卸载后，</font>`<font style="color:rgb(13, 18, 57);">scroll</font>`<font style="color:rgb(13, 18, 57);"> </font><font style="color:rgb(13, 18, 57);">事件仍会触发</font>

+ `<font style="color:rgb(13, 18, 57);">handler</font>`<font style="color:rgb(13, 18, 57);"> 无法释放 → 内存泄漏</font>

### <input>
<input>的value属性

+ 设置了value，那么value的初始值就是你设置的
    - 这里就区别于placeholder了，设置value和placeholder都能设置初始时展示。但是placeholder并没有改变value
+ 如果需要自己控制value，可以通过设置id，然后通过<font style="color:rgb(238, 153, 0);background-color:rgb(245, 242, 240);">document</font><font style="color:rgb(153, 153, 153);background-color:rgb(245, 242, 240);">.</font><font style="color:rgb(221, 74, 104);background-color:rgb(245, 242, 240);">getElementById</font><font style="color:rgb(153, 153, 153);background-color:rgb(245, 242, 240);">(</font><font style="color:rgb(102, 153, 0);background-color:rgb(245, 242, 240);">"myInput"</font><font style="color:rgb(153, 153, 153);background-color:rgb(245, 242, 240);">).</font><font style="color:rgb(0, 0, 0);background-color:rgb(245, 242, 240);">value</font> 来操作
+ onChange属性：焦点离开<input>时执行onchange函数
+ html本身是不支持value属性设置为变量的。vue和react里面支持，这种模式叫做“受控组件”
    - <font style="color:rgb(0, 0, 0);">受控组件中，表单数据是由 React/Vue 组件来管理的。非受控组件中，表单数据由 DOM 节点来处理。</font>
        * <font style="color:rgb(0, 0, 0);">显然，当需要对表单数据有更多更复杂的控制时，需要使用受控组件</font>
    - <font style="color:rgb(0, 0, 0);">react控制input：设置value=一个state，通过onchange函数来更新。</font>
        * <font style="color:rgb(0, 0, 0);">如果只是想设置初始值，则设置defaultValue</font>

```javascript
import React, { useState } from 'react';

function ControlledInput() {
  // 使用useState钩子初始化输入值的状态
  const [inputValue, setInputValue] = useState('');

  // 定义onChange事件处理器来更新状态
  const handleChange = (event) => {
    setInputValue(event.target.value);
  };

  return (
    <input
      type="text"
      value={inputValue} // 输入框的值与state绑定
      onChange={handleChange} // 当输入值变化时调用此函数更新state
    />
  );
}

export default ControlledInput;

```

##### <font style="color:rgb(51, 51, 51);background-color:rgb(247, 249, 253);">过时状态问题</font>
**<font style="color:rgb(27, 27, 27);">闭包</font>**<font style="color:rgb(27, 27, 27);">（closure）是一个函数以及其上下文环境的引用的组合。换而言之，闭包让开发者可以从内部函数访问外部函数的作用域。在 JavaScript 中，闭包会随着函数的创建而被同时创建。</font>

<font style="color:rgb(27, 27, 27);">回调函数常作为闭包，能够在事件触发或异步操作完成后</font>**<font style="color:rgb(27, 27, 27);">访问到其定义时的上下文</font>**<font style="color:rgb(27, 27, 27);">，即使那个上下文在回调实际执行时早已不存在。</font>

<font style="color:rgb(27, 27, 27);"></font>

<font style="color:rgb(27, 27, 27);">在闭包中，可能会访问到过时状态。</font>

```javascript
function Counter() {
  const [count, setCount] = useState(0);
  function add() {
    setTimeout(() => {
      setCount(count + 1);
    }, 3000);
  }
  function preAdd(){
    setTimeout(() => {
      // 根据前一时刻的 count 设置新的 count
      setCount(count => count + 1);
    }, 3000);
  }
  // 监听 count 变化
  useEffect(() => {
    console.log(count)
  }, [count])
  return (
    <>
    Count: {count}
            <button onClick={add}>add</button>
            <button onClick={preAdd}>preAdd</button>
            </>
  );
}
```

<font style="color:rgb(27, 27, 27);">我们首先快速点击 add 按钮三次，三秒后 count 变为 1；然后快速点击 preAdd 三下，三秒后依次出现了 2、3、4。</font>

<font style="color:rgb(27, 27, 27);">why?</font>

+ <font style="color:rgb(27, 27, 27);">state变化引起渲染，每次渲染中有一个state的快照</font>
+ <font style="color:rgb(27, 27, 27);">setTimeout的回调函数是一个闭包，setCount(count + 1)访问的是定义时的count</font>
    - <font style="color:rgb(27, 27, 27);">ps. 在JS的内存管理的垃圾回收是：当变量不再需要的时候，会被销毁，内存空间被回收。所以会在闭包使用完上下文后，再回收</font>

<font style="color:rgb(27, 27, 27);">快速点击 add 按钮三次，三秒后 count 变为 1：</font>

+ 因为点的这3次，count都还没更新（为0），所以这个3个闭包取的都是这个初次渲染的count（0）。而每次更新是在一个新的内存空间里更新count，所以3次回调函数执行时，count都为0。

<font style="color:rgb(27, 27, 27);">快速点击 preAdd 三下，三秒后依次出现了 2、3、4：</font>

+ 因为 setCount(count => count + 1) 他参数是一个函数，会读取当前的count



**当在 回调函数/闭包 中更新state时，要基于当前的state更新，要用setState(state => ...)**



##### 复杂状态管理
<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">useState()</font><font style="color:rgb(37, 41, 51);">用于管理简单状态。对于复杂的状态管理，可以使用</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">useReducer()</font><font style="color:rgb(37, 41, 51);"> hook，它可以自定义多种对state的操作</font>

```javascript
import React, { useReducer } from "react";
import ReactDOM from "react-dom";
import "./styles.css";

const myReducer = (state, action) => {
  switch(action.type) {
    case('countUp'):
      return {
        ...state,
        count: state.count + 1
      }
    default:
      return state
  }
}

function App() {
  const [state, dispatch] = useReducer(myReducer, { count: 0 })

  return (
    <div className="App">
      <button onClick={() => dispatch({ type: 'countUp' })}>
        +1
      </button>
      <p>Count: {state.count}</p>
    </div>
  );
}

const rootElement = document.getElementById("root");
ReactDOM.render(<App />, rootElement);
```

+ 改变ref不会引发渲染，改变state会

```javascript
import { useRef } from 'react';

function MyComponent() {
```

State：

+ 通过<font style="color:rgb(51, 51, 51);"> </font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">@observable</font>`装饰器让state能被mobx追踪

    - state的属性是对象a的话（也就是属性为对象a的引用时），更改对象a的属性，mobx无法观察到。但若对象a声明了observable的话，则可以观察到

Computed：

+ 由state计算得到的值（衍生出来的值）
+ <font style="color:rgb(51, 51, 51);">getter 上使用 </font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">@computed</font>`<font style="color:rgb(51, 51, 51);"> 装饰器来声明</font>

<font style="color:rgb(51, 51, 51);">Actions：</font>

+ <font style="color:rgb(51, 51, 51);">动作是任何用来修改状态的东西</font>
+ <font style="color:rgb(51, 51, 51);"> </font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">@action</font>`<font style="color:rgb(51, 51, 51);"> 用来装饰一个涉及修改state的函数。永远只对</font>**<font style="color:rgb(51, 51, 51);">修改</font>**<font style="color:rgb(51, 51, 51);">状态的函数使用动作。 只执行查找，过滤器等函数</font>**<font style="color:rgb(51, 51, 51);">不</font>**<font style="color:rgb(51, 51, 51);">应该被标记为动作</font>
+ <font style="color:rgb(51, 51, 51);">如果mobx开启了</font>[enforceActions](https://github.com/mobxjs/mobx/blob/gh-pages/docs/refguide/api.md#configure)<font style="color:rgb(51, 51, 51);">，则只能通过 </font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">@action</font>`<font style="color:rgb(51, 51, 51);"> 来修改state（在action函数里面，state=...这样写）。若没开，则可直接在任何地方修改state（能被mobx观察到）</font>
+ 一个例子，ping-pc项目，设计action时可以参考这种思路：

```javascript
// 统一通过这个函数来更新state
@action updateStore(payload: any = {}) {
    const newState = {
      ...payload,
    };
    Object.assign(this.state, newState); // 更新state
}

// 在别的普通函数里调用action
async fetchUserAssetsInfo() {
  //...
  this.updateStore({
    assetsInfo: data.userInfo,
    signInfo: data.userSignInfo,
    signRobotText: data.robotText,
  });
  //...
}

// 对于有些逻辑要处理的state的更新，可以专门给他写个action
@action async updateUserInfo() {
  const [, res1] = await Promise.all<[any, any, any, any, any]>([
    ipcService.call(IPC_COMMAND.UPDATE_USER_INFO, {}),
    ipcService.call(IPC_COMMAND.GET_COUPON_LIST, {}),
    resPositionStore.updateRes(RES_POSITION_CODE_ENUM.USER_INFO_VIP_CARD),
    resPositionStore.updateRes(RES_POSITION_CODE_ENUM.USER_POPOVER_TIP),
    this.fetchUserAssetsInfo(),
  ]);
  if (res1.success) {
    this.state.voucherList = res1.voucherList; // 更新state
  }
}
```

<font style="color:rgb(51, 51, 51);">打包的内容和目的</font>

1. 资源合并：
    - <font style="color:rgb(51, 51, 51);">将多个 JavaScript 和 CSS 文件合并，减少 HTTP 请求的数量，提高页面加载速度。</font>
2. 代码压缩：
    - <font style="color:rgb(51, 51, 51);">对 JavaScript、CSS 等文件进行压缩，去掉不必要的空格、注释等，以减小文件大小，加快加载速度。</font>
3. 模块化：
    - <font style="color:rgb(51, 51, 51);">现代前端开发中，很多库和框架（如 React、Vue 等）使用模块化的方式编写代码。打包工具能够解析这些模块依赖关系，生成一个或多个最终可执行的文件。</font>
4. 处理静态资源：
    - <font style="color:rgb(51, 51, 51);">打包工具通常会处理图像、字体等静态资源，可能将它们复制到输出目录，并生成适当的引用路径。</font>
5. 自动化构建：
    - <font style="color:rgb(51, 51, 51);">打包工具（如 Webpack、Vite、Parcel、Rollup 等）通常集成了自动化构建的功能，可以自动执行压缩、合并、热重载等一系列开发和发布流程。</font>
    - <font style="color:rgb(51, 51, 51);">ps. 当然也可以手动打包，但是效果肯定不如工具</font>
6. 环境变量和配置：
    - <font style="color:rgb(51, 51, 51);">可以根据不同的环境（开发、测试、生产）来配置不同的构建方式。例如，生产环境的构建可以开启更多的优化（如代码压缩、tree shaking 等）。</font>

</font>**<font style="color:rgb(38, 38, 38);">浏览器、node事件循环</font>**
#### <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">浏览器架构里的进程线程</font>
<font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">JavaScript是单线程执行的，指的是一个进程里只有一个主线程</font>



<font style="color:rgb(44, 62, 80);">现代浏览器通常采用多进程架构，其中包括以下主要进程和线程：</font>

1. <font style="color:rgb(44, 62, 80);">浏览器进程（Browser Process）：也称为主进程或渲染进程管理器，负责协调和管理其他进程。它处理用户界面、用户输入、网络请求和文件访问等操作。每个浏览器标签页通常对应一个渲染进程。</font>
2. <font style="color:rgb(44, 62, 80);">渲染进程（Renderer Process）：每个标签页都有一个独立的渲染进程，负责处理页面的渲染和用户交互。渲染进程包含多个线程，包括以下几个主要线程：</font>
    - <font style="color:rgb(44, 62, 80);">主线程（Main Thread）：负责解析HTML、CSS，构建DOM树和渲染页面。</font>
    - <font style="color:rgb(44, 62, 80);">JavaScript引擎线程：负责执行JavaScript代码。</font>
    - <font style="color:rgb(44, 62, 80);">事件线程（Event Thread）：处理用户交互事件，如鼠标点击、键盘输入等。</font>
    - <font style="color:rgb(44, 62, 80);">布局线程（Layout Thread）：负责计算页面布局。</font>
    - <font style="color:rgb(44, 62, 80);">绘制线程（Paint Thread）：将页面内容绘制到屏幕上。</font>
3. <font style="color:rgb(44, 62, 80);">GPU进程（GPU Process）：负责处理页面中需要使用图形加速的任务，如绘制3D效果、视频解码等。它与渲染进程协同工作，通过硬件加速提高图形性能。</font>
4. <font style="color:rgb(44, 62, 80);">网络进程（Network Process）：负责处理网络请求和响应，包括DNS解析、HTTP请求等。它与渲染进程分离，提供独立的网络功能。</font>
5. <font style="color:rgb(44, 62, 80);">插件进程（Plugin Process）：负责运行浏览器插件，如Flash插件。由于插件的安全性和稳定性问题，现代浏览器逐渐减少对插件的支持。</font>

<font style="color:rgb(44, 62, 80);">js同步代码的执行、setTimeout（的回调函数）、setInterval（的回调函数）、setImmediate（的回调函数）（浏览器暂时不支持，只有IE10支持）、I/O（的回调函数）、UI Rendering。</font>

<font style="color:rgb(44, 62, 80);">Process.nextTick（Node独有）、Promise、Object.observe(废弃)、MutationObserver（具体使用方式查看 </font>[<font style="color:rgb(35, 93, 200);">这里(opens new window)</font>](https://javascript.ruanyifeng.com/dom/mutationobserver.html)<font style="color:rgb(44, 62, 80);">）</font>



#### <font style="color:rgb(44, 62, 80);">浏览器事件循环</font>
<font style="color:rgb(44, 62, 80);">Javascript 有一个 main thread 主线程和 call-stack 调用栈(执行栈)</font>

<font style="color:rgb(44, 62, 80);">所有的任务都会被放到调用栈等待主线程执行</font>



<font style="color:rgb(44, 62, 80);">同步任务会在</font>**<font style="color:rgb(44, 62, 80);">调用栈</font>**<font style="color:rgb(44, 62, 80);">中按照顺序等待主线程依次执行</font>

<font style="color:rgb(44, 62, 80);">异步任务会在异步任务有了结果后（例如 setTimeout 的等待时间结束），将注册的回调函数放入</font>**<font style="color:rgb(44, 62, 80);">任务队列</font>**<font style="color:rgb(44, 62, 80);">中等待主线程空闲的时候（调用栈被清空），被读取到栈内等待主线程的执行。</font>

+ <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">分为宏任务队列和微任务队列</font>



<font style="color:rgb(44, 62, 80);">任务的具体执行顺序：（</font>**<font style="color:rgb(44, 62, 80);">执行一个宏任务后，执行完微任务队列</font>**<font style="color:rgb(44, 62, 80);">）</font>

1. <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">执行所有的同步代码（相当于一个宏任务）</font>
2. <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">清空微任务队列</font>
3. <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">从宏任务队列中拿一个宏任务执行</font>
4. <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">清空微任务队列</font>
5. <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">循环3-4</font>

#### <font style="color:rgb(31, 41, 55);background-color:rgb(249, 250, 251);">Node事件循环</font>
<img src="https://intranetproxy.alipay.com/skylark/lark/0/2025/png/88656377/1745897931662-7fecc0fd-cee2-43f0-934b-afea343284a1.png" width="674" title="" crop="0,0,1,1" id="CuLvt" class="ne-image" style="color: rgb(44, 62, 80); font-size: 16px">

<font style="color:rgb(44, 62, 80);">Node.js 采用 V8 作为 js 的解析引擎，而 I/O 处理方面使用了自己设计的 libuv</font>

+ <font style="color:rgb(44, 62, 80);">libuv 是一个基于事件驱动的跨平台抽象层，封装了不同操作系统一些底层特性，对外提供统一的 API（时间，非阻塞的网络，异步文件操作，子进程等等），Event Loop 也是它里面的实现。</font>

<font style="color:rgb(44, 62, 80);">Node.js的运行机制如下:</font>

+ <font style="color:rgb(44, 62, 80);">V8 引擎解析 JavaScript 脚本。</font>
+ <font style="color:rgb(44, 62, 80);">解析后的代码，调用 Node API。</font>
+ <font style="color:rgb(44, 62, 80);">libuv 库负责 Node API 的执行。它将不同的任务分配给不同的线程，形成一个 Event Loop（事件循环），以异步的方式将任务的执行结果返回给 V8 引擎。</font>
+ <font style="color:rgb(44, 62, 80);">V8 引擎再将结果返回给用户。</font>



<img src="https://intranetproxy.alipay.com/skylark/lark/0/2025/png/88656377/1745897931311-26f9fa08-9696-49db-92d3-eae55468284a.png" width="924" title="" crop="0,0,1,1" id="IvxxD" class="ne-image" style="color: rgb(44, 62, 80); font-size: 16px">

<font style="color:rgb(44, 62, 80);">Node 的task（宏任务）执行分为几个阶段：</font>

1. <font style="color:rgb(44, 62, 80);">timers定时器：本阶段执行已经安排的 setTimeout() 和 setInterval() 的回调函数。</font>
2. <font style="color:rgb(44, 62, 80);">pending callbacks待定回调：执行延迟到下一个循环迭代的 I/O 回调。</font>
3. <font style="color:rgb(44, 62, 80);">idle, prepare：仅系统内部使用。</font>
4. <font style="color:rgb(44, 62, 80);">poll 轮询：检索新的 I/O 事件;执行与 I/O 相关的回调（几乎所有情况下，除了关闭的回调函数，它们由计时器和 setImmediate() 排定的之外），其余情况 node 将在此处阻塞。</font>
5. <font style="color:rgb(44, 62, 80);">check 检测：setImmediate() 回调函数在这里执行。</font>
6. <font style="color:rgb(44, 62, 80);">close callbacks 关闭的回调函数：一些准备关闭的回调函数，如：socket.on('close', ...)。</font>



**<font style="color:rgb(31, 35, 40);">微任务和宏任务在Node的执行顺序</font>**

<font style="color:rgb(31, 35, 40);">Node 10以前：</font>

+ <font style="color:rgb(31, 35, 40);">执行完一个阶段的所有宏任务（同步代码的执行也相当于一个阶段的宏任务）</font>
+ <font style="color:rgb(31, 35, 40);">执行完nextTick队列</font>
+ <font style="color:rgb(31, 35, 40);">执行完微任务队列</font>

<font style="color:rgb(31, 35, 40);">Node 11以后： 和浏览器的行为统一了，都是每执行一个宏任务就执行完微任务队列。</font>
